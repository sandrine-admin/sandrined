    Notes techniques sur mes projets et déploiements.

    Contexte : J'avais besoin d'accéder à mes machines Windows Server 2025 (JMLDC01, JMLDC02) via RDP depuis n'importe où, sans exposer le port 3389 directement sur Internet. Apache Guacamole permet d'accéder au bureau à distance via un simple navigateur web.

    Infrastructure :

    • Déploiement via Docker Compose sur un conteneur LXC Proxmox
    • Stack composée de deux conteneurs : guacamole et guacd
    • Backend de stockage : fichier de configuration XML (mode sans base de données)
    • Exposition publique via Cloudflare Tunnel sous rdp.sandrinede.com/guacamole

    Ce que j'ai configuré :

    • Connexion RDP vers JMLDC01 et JMLDC02 avec authentification domaine
    • Lancement automatique de l'interface WPF GUI JML au démarrage de la session RDP
    • Configuration du tunnel Cloudflare pour router le trafic HTTPS vers Guacamole

    Problèmes rencontrés :

    • Le chemin /guacamole dans l'URL nécessitait une configuration spécifique côté Cloudflare Tunnel
    • Intérêt d'évoluer vers un backend MySQL pour gérer les utilisateurs plus facilement (prévu)

    Résultat : Accès RDP sécurisé depuis n'importe quel navigateur, sans VPN, sans exposition de port RDP.

    Outils utilisés :  Apache Guacamole Docker Compose Proxmox LXC Cloudflare Tunnel Windows Server 2025

    Contexte : Dans le cadre de mon TFE, j'ai déployé Wazuh pour monitorer l'infrastructure et détecter les événements de sécurité sur mes contrôleurs de domaine.

    Infrastructure :

    • Installation de Wazuh sur une VM Ubuntu dédiée dans Proxmox
    • Agents Wazuh installés sur JMLDC01 et JMLDC02 (Windows Server 2025)
    • Dashboard exposé via Cloudflare Tunnel sous wazuh.sandrinede.com
    • Index configuré : wazuh-states-vulnerabilities-*

    Ce que j'ai configuré :

    • Création d'un service systemd wazuh-start.service pour le bon ordre de démarrage
    • Configuration des dashboards de monitoring dans l'interface web
    • Connexion des agents Windows au manager Wazuh

    Problèmes rencontrés :

    • L'index wazuh-alerts-* restait vide à cause d'une incompatibilité de version avec Filebeat
    • Ordre de démarrage des services incorrect au boot — résolu avec un service systemd personnalisé

    Résultat : Dashboard SIEM fonctionnel avec visibilité sur les vulnérabilités et les événements système.

    Outils utilisés :  Wazuh Ubuntu Proxmox Cloudflare Tunnel Windows Server 2025

    Contexte : Pour exposer mes services homelab sur Internet sans ouvrir de ports sur ma box, j'utilise Cloudflare Tunnel — une solution sécurisée qui crée un tunnel chiffré entre mes services et Cloudflare.

    Services exposés :

    • portail-rh.sandrinede.com — Portail web JML
    • wazuh.sandrinede.com — Dashboard SIEM
    • rdp.sandrinede.com — Apache Guacamole
    • infra.sandrinede.com — Dashboard Proxmox temps réel

    Ce que j'ai configuré :

    • Installation du daemon cloudflared sur les conteneurs LXC concernés
    • Création des tunnels et règles d'ingress dans le dashboard Cloudflare
    • Certificats HTTPS automatiques via Cloudflare (pas besoin de Let's Encrypt)

    Avantages :

    • Aucun port ouvert sur la box — sécurité maximale
    • HTTPS automatique sur tous les sous-domaines
    • Protection DDoS incluse

    Outils utilisés :  Cloudflare Tunnel cloudflared Proxmox LXC DNS Cloudflare

    Contexte : Mon TFE porte sur l'automatisation du cycle de vie des comptes utilisateurs (Joiners-Movers-Leavers) dans un environnement hybride Active Directory et Entra ID (Azure AD).
