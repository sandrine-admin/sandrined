Et la suite ?

Notes techniques sur mes projets et déploiements.

Contexte : J'avais besoin d'accéder à mes machines Windows Server 2025 via RDP depuis n'importe où, sans exposer le port 3389 sur Internet. Apache Guacamole permet d'accéder au bureau à distance via un simple navigateur web.

Infrastructure :

  • Déploiement via Docker Compose sur un conteneur LXC Proxmox
  • Stack composée de deux conteneurs : guacamole et guacd
  • Backend de stockage : fichier de configuration XML (mode sans base de données)
  • Exposition publique via Cloudflare Tunnel sous rdp.sandrinede.com/guacamole

Ce que j'ai configuré :

  • Connexion RDP vers JMLDC01 et JMLDC02 avec authentification domaine
  • Lancement automatique de l'interface WPF GUI JML au démarrage de la session RDP
  • Configuration du tunnel Cloudflare pour router le trafic HTTPS vers Guacamole

Problèmes rencontrés :

  • Le chemin /guacamole nécessitait une configuration spécifique côté Cloudflare Tunnel
  • Intérêt d'évoluer vers un backend MySQL pour gérer les utilisateurs plus facilement (prévu)

Résultat : Accès RDP sécurisé depuis n'importe quel navigateur, sans VPN, sans exposition de port RDP.

Outils utilisés :  Apache Guacamole Docker Compose Proxmox LXC Cloudflare Tunnel Windows Server 2025

Contexte : Dans le cadre de mon TFE, j'ai déployé Wazuh pour monitorer l'infrastructure et détecter les événements de sécurité sur mes contrôleurs de domaine.

Infrastructure :

  • Installation de Wazuh sur une VM Ubuntu dédiée dans Proxmox
  • Agents Wazuh installés sur JMLDC01 et JMLDC02 (Windows Server 2025)
  • Dashboard exposé via Cloudflare Tunnel sous wazuh.sandrinede.com
  • Index configuré : wazuh-states-vulnerabilities-*

Ce que j'ai configuré :

  • Création d'un service systemd wazuh-start.service pour le bon ordre de démarrage
  • Configuration des dashboards de monitoring dans l'interface web
  • Connexion des agents Windows au manager Wazuh

Problèmes rencontrés :

  • L'index wazuh-alerts-* restait vide à cause d'une incompatibilité de version avec Filebeat
  • Ordre de démarrage des services incorrect au boot — résolu avec un service systemd personnalisé

Résultat : Dashboard SIEM fonctionnel avec visibilité sur les vulnérabilités et les événements système.

Outils utilisés :  Wazuh Ubuntu Proxmox Cloudflare Tunnel Windows Server 2025

Contexte : Pour exposer mes services homelab sur Internet sans ouvrir de ports sur ma box, j'utilise Cloudflare Tunnel — une solution sécurisée qui crée un tunnel chiffré entre mes services et Cloudflare.

Services exposés :

  • portail-rh.sandrinede.com — Portail web JML
  • wazuh.sandrinede.com — Dashboard SIEM
  • rdp.sandrinede.com — Apache Guacamole
  • infra.sandrinede.com — Dashboard Proxmox temps réel

Ce que j'ai configuré :

  • Installation du daemon cloudflared sur les conteneurs LXC concernés
  • Création des tunnels et règles d'ingress dans le dashboard Cloudflare
  • Certificats HTTPS automatiques via Cloudflare

Avantages :

  • Aucun port ouvert sur la box — sécurité maximale
  • HTTPS automatique sur tous les sous-domaines
  • Protection DDoS incluse

Outils utilisés :  Cloudflare Tunnel cloudflared Proxmox LXC DNS Cloudflare

Glossaire

Définitions des termes techniques utilisés dans mes projets.

Proxmox VE
Hyperviseur open-source basé sur Linux permettant de gérer des machines virtuelles (VMs) et des conteneurs LXC depuis une interface web.
VM
Machine Virtuelle — émulation complète d'un ordinateur physique, avec son propre OS, isolée du système hôte.
LXC
Linux Container — conteneur léger qui partage le noyau Linux de l'hôte. Plus léger qu'une VM, idéal pour les services Linux simples.
Docker
Plateforme de conteneurisation qui permet d'empaqueter une application et ses dépendances dans un conteneur portable et isolé.
Docker Compose
Outil permettant de définir et gérer plusieurs conteneurs Docker via un fichier YAML, simplifiant le déploiement de stacks applicatives.
Hyperviseur
Logiciel qui permet de faire tourner plusieurs systèmes d'exploitation sur une même machine physique en les isolant les uns des autres.
Snapshot
Capture de l'état d'une VM ou d'un conteneur à un instant précis, permettant de revenir en arrière en cas de problème.
Stack
Ensemble de logiciels/services qui fonctionnent ensemble pour faire tourner une application. Ex: la stack Guacamole = conteneur guacamole + conteneur guacd.
Daemon
Programme qui tourne en arrière-plan en permanence sur un système Linux, sans interface graphique. Ex: cloudflared maintient le tunnel Cloudflare actif.
DNS
Domain Name System — système qui traduit les noms de domaine (ex: sandrinede.com) en adresses IP compréhensibles par les machines.
DHCP
Dynamic Host Configuration Protocol — protocole qui attribue automatiquement une adresse IP aux appareils qui se connectent au réseau.
VPN
Virtual Private Network — tunnel chiffré entre deux points sur Internet, permettant de sécuriser les communications ou d'accéder à un réseau privé à distance.
Tunnel
Connexion chiffrée entre deux points qui encapsule le trafic réseau. Utilisé ici via Cloudflare pour exposer des services sans ouvrir de ports.
VLAN
Virtual LAN — segmentation logique d'un réseau physique en plusieurs réseaux virtuels isolés, pour des raisons de sécurité ou d'organisation.
Pare-feu
Système de sécurité qui filtre le trafic réseau entrant et sortant selon des règles définies, pour protéger le réseau des accès non autorisés.
Proxy
Serveur intermédiaire entre un client et un serveur, qui peut filtrer, mettre en cache ou anonymiser les requêtes réseau.
RDP
Remote Desktop Protocol — protocole Microsoft permettant de prendre le contrôle d'un bureau Windows à distance via le réseau.
SSH
Secure Shell — protocole chiffré permettant d'accéder à distance à un terminal Linux/Unix de manière sécurisée.
Port
Point d'entrée numérique sur un serveur pour un service spécifique. Ex: port 80 pour HTTP, 443 pour HTTPS, 3389 pour RDP. Cloudflare Tunnel évite d'ouvrir ces ports.
Sous-domaine
Préfixe ajouté à un nom de domaine principal. Ex: wazuh.sandrinede.com, infra.sandrinede.com sont des sous-domaines de sandrinede.com.
Pi-hole
Serveur DNS local qui bloque les publicités et domaines malveillants pour tous les appareils du réseau, sans configuration sur chaque appareil.
Active Directory
Service d'annuaire Microsoft qui centralise la gestion des utilisateurs, ordinateurs et ressources d'un réseau Windows (authentification, droits, politiques).
Entra ID
Anciennement Azure Active Directory — service d'identité cloud de Microsoft qui gère les accès aux applications et ressources cloud.
Domaine
Ensemble d'ordinateurs et d'utilisateurs gérés centralement par un Active Directory. Tous les objets partagent la même base de données d'annuaire.
Contrôleur de domaine
Serveur Windows qui héberge Active Directory et gère l'authentification et les politiques pour tous les objets du domaine.
OU
Organizational Unit — unité organisationnelle dans Active Directory permettant de regrouper et organiser les objets pour y appliquer des GPO.
GPO
Group Policy Object — ensemble de règles et paramètres appliqués automatiquement aux utilisateurs et ordinateurs d'un domaine Active Directory.
JML
Joiners-Movers-Leavers — processus RH et IT de gestion du cycle de vie des comptes : création à l'arrivée, modification en cas de changement de poste, désactivation au départ.
LDAP
Lightweight Directory Access Protocol — protocole standard pour accéder et modifier les annuaires comme Active Directory.
PowerShell
Langage de script et shell de Microsoft, très utilisé pour l'automatisation des tâches d'administration Windows et Active Directory.
WPF
Windows Presentation Foundation — framework Microsoft pour créer des interfaces graphiques (GUI) pour les applications Windows.
SIEM
Security Information and Event Management — système qui collecte, analyse et corrèle les logs et événements de sécurité de l'infrastructure pour détecter les menaces.
Wazuh
Solution SIEM open-source qui surveille les systèmes, détecte les intrusions, analyse les vulnérabilités et génère des alertes de sécurité.
IAM
Identity and Access Management — ensemble de processus et outils pour gérer les identités numériques et contrôler qui a accès à quoi dans un système.
MFA
Multi-Factor Authentication — méthode d'authentification qui exige au moins deux preuves d'identité (ex: mot de passe + code SMS) pour accéder à un compte.
Zero Trust
Modèle de sécurité qui ne fait confiance à aucun utilisateur ou appareil par défaut, même à l'intérieur du réseau. Chaque accès doit être vérifié.
Chiffrement
Transformation des données en un format illisible sans clé de déchiffrement, pour protéger les informations en transit ou au repos.
HTTPS
HTTP Sécurisé — version chiffrée du protocole web HTTP, utilisant TLS pour protéger les échanges entre le navigateur et le serveur.
Certificat SSL/TLS
Fichier numérique qui authentifie l'identité d'un site web et chiffre les communications, matérialisé par le cadenas dans le navigateur.
IaaS
Infrastructure as a Service — modèle cloud où le fournisseur gère le matériel et la virtualisation. L'utilisateur gère l'OS, les applications et les données. Ex: AWS EC2, Azure VM.
PaaS
Platform as a Service — modèle cloud où le fournisseur gère l'infrastructure et l'OS. L'utilisateur se concentre uniquement sur son application. Ex: Heroku, Azure App Service.
SaaS
Software as a Service — logiciel hébergé et géré entièrement par le fournisseur, accessible via navigateur. L'utilisateur ne gère rien. Ex: Gmail, Microsoft 365, Zoom.
API
Application Programming Interface — interface qui permet à deux applications de communiquer entre elles. Ex: l'API Proxmox permet à Grafana de récupérer les données de l'infrastructure.
API REST
Type d'API qui fonctionne via HTTP. Elle utilise des requêtes GET (lire), POST (créer), PUT (modifier) et DELETE (supprimer) pour interagir avec un service distant.
Token
Clé d'accès temporaire utilisée pour s'authentifier auprès d'une API sans utiliser un mot de passe. Ex: le token read-only Proxmox utilisé par Grafana.
Azure
Plateforme cloud de Microsoft offrant des services d'infrastructure, d'identité (Entra ID), de sécurité et d'applications à l'échelle mondiale.
AWS
Amazon Web Services — plateforme cloud d'Amazon, leader du marché, offrant plus de 200 services (calcul, stockage, base de données, IA...).
Cloudflare
Service réseau mondial offrant DNS, CDN, protection DDoS et tunnels sécurisés. Utilisé ici pour exposer les services du homelab sans ouvrir de ports.
CDN
Content Delivery Network — réseau de serveurs répartis géographiquement pour distribuer le contenu web au plus près des utilisateurs et réduire la latence.
Microsoft Graph API
API REST de Microsoft permettant d'interagir avec les services Microsoft 365 et Entra ID (gestion des utilisateurs, groupes, mails...) via des requêtes HTTP.
Entra Connect
Outil Microsoft qui synchronise les comptes d'un Active Directory on-premises vers Entra ID (Azure AD), permettant une gestion d'identité hybride.
Tenant
Instance dédiée des services Microsoft 365 / Entra ID d'une organisation. Chaque entreprise ou lab a son propre tenant avec ses utilisateurs et ressources.
EC2
Elastic Compute Cloud — service AWS qui permet de louer des serveurs virtuels (instances) dans le cloud. C'est le service IaaS de base d'AWS.
S3
Simple Storage Service — service de stockage objet AWS permettant de stocker et récupérer n'importe quelle quantité de données. Très utilisé pour les sauvegardes, sites statiques et fichiers.
IAM (AWS)
Identity and Access Management — service AWS qui gère les utilisateurs, groupes et permissions d'accès aux ressources AWS. Permet d'appliquer le principe du moindre privilège.
Region
Zone géographique où AWS dispose de data centers. Ex: eu-west-1 (Irlande), eu-central-1 (Francfort). Chaque région est indépendante pour la résilience et la conformité des données.
Availability Zone
Zone de disponibilité — data center physiquement séparé à l'intérieur d'une région AWS. Chaque région en a minimum 3, pour garantir la haute disponibilité.
VPC
Virtual Private Cloud — réseau privé virtuel isolé dans AWS où tu déploies tes ressources. Permet de contrôler les adresses IP, sous-réseaux et règles de sécurité réseau.
Security Group
Pare-feu virtuel AWS qui contrôle le trafic entrant et sortant des instances EC2. Fonctionne comme une liste de règles autorisant ou bloquant des ports et protocoles.
Lambda
Service AWS de Functions as a Service (FaaS) — exécute du code sans gérer de serveur. Le code ne tourne que lors d'un déclenchement et la facturation est à la milliseconde.
RDS
Relational Database Service — service AWS de base de données managée (MySQL, PostgreSQL, SQL Server...). AWS gère les sauvegardes, mises à jour et haute disponibilité.
CloudWatch
Service de monitoring et logs AWS. Collecte les métriques des services AWS, génère des alertes et stocke les logs applicatifs.
CloudTrail
Service AWS qui enregistre toutes les actions effectuées sur le compte AWS (qui a fait quoi, quand). Essentiel pour l'audit de sécurité.
SQS
Simple Queue Service — service de file de messages AWS permettant à des applications de communiquer de manière asynchrone. Premier service AWS lancé publiquement en 2004.
Route 53
Service DNS managé d'AWS permettant de gérer les noms de domaine et router le trafic vers les ressources AWS ou externes.
Elastic Beanstalk
Service PaaS d'AWS qui déploie automatiquement une application (Node.js, Python, Java...) sans gérer l'infrastructure sous-jacente.
Pay-as-you-go
Modèle de facturation AWS où tu paies uniquement ce que tu consommes, sans engagement. Pas de serveur à acheter, pas de contrat minimum.
Free Tier
Niveau gratuit AWS offrant un accès limité à certains services pendant 12 mois (ou indéfiniment pour certains). Idéal pour apprendre et tester sans frais.
ARN
Amazon Resource Name — identifiant unique d'une ressource AWS. Format : arn:aws:service:region:account-id:resource. Utilisé dans les politiques IAM et configurations.
CLI AWS
Interface en ligne de commande pour interagir avec les services AWS depuis un terminal, sans passer par la console web.
Linux
Système d'exploitation open-source basé sur le noyau Linux. Très utilisé pour les serveurs, l'infrastructure et le cloud. La majorité des serveurs web et conteneurs tournent sous Linux.
Ubuntu
Distribution Linux populaire basée sur Debian, très utilisée pour les serveurs. Utilisée ici pour les VMs Wazuh et certains conteneurs LXC du homelab.
Bash
Shell et langage de script standard sous Linux. Permet d'automatiser des tâches, gérer des fichiers et exécuter des commandes en séquence.
Systemd
Système d'initialisation et gestionnaire de services Linux. Permet de démarrer, arrêter et surveiller des services au boot. Utilisé ici pour créer le service wazuh-start.service.
Service systemd
Fichier de configuration (.service) qui définit comment un programme doit être lancé par systemd au démarrage du système, avec ses dépendances et son ordre de démarrage.
Terminal
Interface en ligne de commande permettant d'interagir avec le système d'exploitation via du texte. Indispensable pour administrer des serveurs Linux à distance.
Cron
Planificateur de tâches Linux qui exécute automatiquement des scripts ou commandes à des horaires définis. Équivalent Linux du Planificateur de tâches Windows.
Package
Logiciel précompilé installable via un gestionnaire de paquets (apt, yum...). Ex: sudo apt install nginx installe le serveur web Nginx.
apt
Advanced Package Tool — gestionnaire de paquets des distributions Debian/Ubuntu. Permet d'installer, mettre à jour et supprimer des logiciels facilement.
Root
Compte super-administrateur sous Linux avec tous les droits sur le système. Équivalent de l'administrateur sous Windows. À utiliser avec précaution.
sudo
Commande Linux permettant d'exécuter une commande avec les droits root sans se connecter en tant que root. Permet de tracer qui a fait quoi.
Script
Fichier texte contenant une série de commandes exécutées automatiquement. En Bash sous Linux ou en PowerShell sous Windows pour automatiser des tâches répétitives.
Log
Fichier qui enregistre chronologiquement les événements d'un système ou application. Indispensable pour diagnostiquer des problèmes et surveiller la sécurité.
grep
Commande Linux pour rechercher du texte dans des fichiers ou la sortie d'une commande. Très utilisée pour filtrer des logs. Ex: cat syslog | grep error.
Pipe
Opérateur | qui redirige la sortie d'une commande vers l'entrée d'une autre. Permet de chaîner des commandes. Ex: cat fichier.log | grep error | wc -l.
Home Assistant
Plateforme domotique open-source qui centralise et automatise les équipements connectés de la maison. Tourne ici sur un Raspberry Pi.
Raspberry Pi
Nano-ordinateur low-cost et basse consommation, très utilisé pour les projets DIY, la domotique et l'apprentissage. Idéal pour faire tourner Home Assistant en permanence.
Intégration
Dans Home Assistant, module qui connecte un appareil ou service externe (Philips Hue, Spotify, météo...) pour le contrôler depuis l'interface.
Automation
Règle dans Home Assistant qui déclenche automatiquement une action quand une condition est remplie. Ex: allumer les lumières quand le soleil se couche.
Dashboard
Interface visuelle personnalisable dans Home Assistant qui affiche l'état des équipements et permet de les contrôler.
HACS
Home Assistant Community Store — gestionnaire de plugins communautaires pour Home Assistant. Permet d'installer des thèmes, cartes et intégrations non officielles comme Mushroom Cards.
Mushroom Cards
Collection de cartes UI modernes et épurées pour le dashboard Home Assistant, installables via HACS.
IoT
Internet of Things (Internet des Objets) — ensemble des appareils physiques connectés à Internet qui collectent et échangent des données. Ex: ampoules connectées, capteurs de température.
Uptime Kuma
Outil de monitoring open-source auto-hébergé qui surveille la disponibilité des services et envoie des alertes en cas de panne.
